# 住まいのダニ・アレルギー診断（/dani/）— デザイントークン（実装前に確定・自己批評済み）

kurashi-carte.jp の**同一ドメイン下に置く「別物サイト」**として自己完結構築する（BOSS指示 2026-07-06）。
共有ファイル（母屋 index.html・root config.json・root design-tokens.md 等）は**並行構築中の「洗濯・水回り診断」
セッションの領分**につき一切触れない。本サイトの一切は `/dani/` 配下で完結する。

案件固有の視覚アイデンティティ。姉妹・同居サイトのいずれとも**色・書体・モチーフを別系統**にし、
共有するのは構造の思想（config駆動・GAS・インラインSVG・GEO・「カルテ＝診断票の発行」）だけ。

## コンセプト（1行 ＝ 全工程を縛る正典）
> 住まいを部屋ごとに見立て、ダニの居どころを一枚の「住まいの健康診断票（ホームカルテ）」にし、
> **わが家のダニリスク指数**を“天気予報”のようにやさしく発行し、あなたの住まいを継続的に見守るメディア。
> （※リスクの目安であり、医療診断でも害虫の駆除保証でもない ＝ 薬機法・景表法の道義的配慮）

## やらないことリスト（暴走の制御 ／ compliance の差し戻し基準）
- ダニの拡大写真・気持ち悪い実写虫・恐怖訴求の赤黒（＝害虫駆除サイトの定番）を**使わない**。
- 「駆除・退治・殺す・アレルギーが治る・症状が改善」等の断定を**書かない**（**捕獲**＝実機能表現に留める）。
- クリニック水色一辺倒／メンズ黒金／before-after／他社比較・ランキングを**使わない**（比較しない・並べない）。
- 姉妹 karada-carte（緑×朱印・明朝×Georgia）／同居 洗濯診断（青緑×珊瑚×ミント・丸ゴ・水滴ゲージ・泡）の
  視覚を**リサイクルしない**。同一ドメインでも“双子”に見せない。
- 可愛さに振り切って安っぽくしない（＝「予報カード＝診断票」の記録性で信頼を担保）。

## 差別化ゲート（非競合フィールドからの借用）
ダニ・害虫・アレルギー系サイトの典型（恐怖の虫アップ／赤黒／クリニック水色）を回避するため、
**「天気予報・生活指数（花粉情報・洗濯指数・熱中症指数）の視覚言語」を借用**する。
→ ダニ対策を“怖い駆除”でなく“毎日の暮らしの予報”に翻訳。指数ゲージ・5段階予報・季節（梅雨/夏）・
   お天気マスコットで、**可愛く・親しみやすく・目を引く**を、信頼（カルテ＝診断票）と両立。

## シグネチャー要素（記憶される唯一の独自要素）
**「ダニ予報カード ＋ わが家のダニリスク指数（0–100）」の発行体験。**
診断の最後に、天気予報カードの趣で —
1. **わが家のダニリスク指数**（0–100・特大の指数数字＋針ゲージ、カウントアップ演出）
2. **5段階予報**（ひかえめ／ふつう／多め／要注意／繁殖ピーク ＝ 断定を避けた目安語）
3. **お天気マスコット「ひなたちゃん」**（擬人化した“おひさま布団”。指数で表情が5段階に変わる
   ＝一目で予報が伝わる／SNS映え）。※味方＝清潔・快適の象徴。敵＝ダニは可愛くしない（抽象ドット）。
4. **部屋別リスクマップ**（寝室・寝具・リビング・クローゼット・車）＋**部屋別の必要枚数“処方”**（客単価最大化）
5. カルテ番号・発行日・**季節バッジ**（梅雨〜夏＝ピーク）
——を持つ一枚の“住まいの診断票”。周囲（診断フロー）は静かな藍と麻色で抑え、確定の瞬間に
指数の**アンバー〜ローズ針**と**マスコット**を最も強く効かせる＝大胆さの一点集中。印刷/保存/SNSシェアに耐える。

## カラー（名前付き・6色＋2機能色）
「よく眠る寝室の藍夜 × 朝の光（天日干しのアンバー）× 清潔な麻シーツの温白 × 予報の針」。
ダニ系の赤黒・クリニック水色一辺倒を外し、karada（緑/朱）・洗濯（青緑/珊瑚/ミント）とも重ならない、
安眠と朝日の暖冷ミックス。**緑・ミント・珊瑚は使わない**（同居サイトとの差別化を機械的に担保）。

| 変数 | HEX | 役割 |
|---|---|---|
| `--indigo`   | `#29396A` | 主構造色。深い藍夜（安眠の寝室・夜・信頼）＝ヘッダ/フッタ/見出し/濃地 |
| `--indigo-2` | `#3A4E86` | 中藍。グラデ・帯 |
| `--sky`      | `#5E8CCB` | 朝の空色。リンク・副次アクセント（藍より明るい“予報”の抜け感） |
| `--linen`    | `#FCF8F1` | 麻シーツの温白＝ページ地色。※クリーム #F4F1EA より白く、AIデフォルト回避 |
| `--linen-2`  | `#F1EBDD` | セクション/カード地。薄い麻色 |
| `--line`     | `#E6DECE` | 診断票の薄い方眼罫線（1px） |
| `--amber`    | `#EEA23C` | **署名の暖色**。おひさま・天日干し・朝日・指数の陽（快適/CTA/マスコット）。karada朱・洗濯珊瑚と別 |
| `--rose`     | `#C7677A` | 高リスク“繁殖ピーク”のやさしい注意色（ダスティローズ。危険断定の赤を避ける） |
| `--ink`      | `#2A2E3A` | 本文（藍寄りの炭。純黒を避ける） |
| `--muted`    | `#6E7180` | 注記・副次テキスト |
| `--ok` `#4E8C6A`（極控えめ・状態のみ）/ `--alert` `#C7677A`（＝rose） | 状態色 |

- 指数ゲージは **sky→amber→rose** の連続グラデ（低リスク=晴れの空色〜高リスク=蒸し暑いローズ）＝体感の天気色。
  ※ミント/珊瑚を避けるため中間はアンバー。緑系は使わない。

## タイポグラフィ（和文・実在フォント／外部CDN無し・システム同梱で完結）
karada（明朝×Georgia）・洗濯（丸ゴ×Trebuchet）と**別系統**に。信頼の予報ダッシュボード＋暮らしの親しみ。
セキュリティ基準（外部JS/CDNを避けローカル化）に従い、システム同梱書体のみ。

- 見出し（Display）: `"Yu Gothic UI","BIZ UDPGothic","Hiragino Kaku Gothic ProN","Meiryo",sans-serif`（700–800）
  — UDゴシック寄り＝誠実・少し幾何学的（明朝の格式でも丸ゴの甘さでもない“予報／暮らし”の中庸）
- 本文（Body）: `system-ui,"Hiragino Kaku Gothic ProN","Yu Gothic UI","Meiryo",sans-serif` — 高可読性
- 指数・数値（Index/カルテ番号）: `"Bahnschrift","DIN Alternate","Yu Gothic UI",sans-serif`＋`tabular-nums`
  — **天気予報の数字はゴシック**。等幅で“指数ダッシュボード”感（karadaのGeorgiaセリフ数字と差別化）。
- ラテン小ラベル（HOME CARTE / DANI FORECAST）は字間広め。
- 可愛さは「マスコット＋アンバー＋角丸カード＋やわらかい影」で作る（＝丸ゴ書体に頼らない＝洗濯との差別化）。

## レイアウト（すべて `/dani/` 配下で完結）
- `/dani/`（本体トップ＝この別物サイトの顔）: ヒーロー→GEO直接回答→部屋別5ステップ診断→
  **ダニ予報カード発行（指数＋マスコット＋部屋処方）**→部屋別の必要枚数オファー（Apros ダニ取りシート・定期便）→
  購入後の再診断/回遊導線→季節→しくみ→FAQ。max 880px。
- `/dani/shisuu/`: 固有指標「わが家のダニリスク指数」の定義・算出の考え方の解説（AI一次ソース化）。Article。
- `/dani/thanks.html`: 送信/購入後の再診断案内・回遊（別部屋追加/リピート/グループ他商品）。URLパラメータで初回スコア受け渡し。
- ヘッダは「← 暮らしのカルテ」で母屋 `/`（洗濯セッションが構築するハブ）へリンク＝孤立させない。
- 予報カードは方眼罫・カルテ番号・季節バッジ・部屋マップを持つ“紙面”。print CSS で保存に耐える。

## 透かし画像／背景レイヤー（可愛く派手に・BOSS必須要件）
全て**インラインSVG**（外部ホットリンク皆無＝セキュリティ基準）。淡いモチーフを背景に敷き奥行きを出す：
- `wm-dots`: 空気中のホコリ/ハウスダストを柔らかい点描にした極薄ドット（body背景・方眼罫と重ねる）
- `wm-madori`: 間取り線（住まい）の極薄ラインアート（ヒーロー背景）
- `wm-sun`: 朝日／光条の極薄グラデ（予報カード上部）
- マスコット「ひなたちゃん」（おひさま布団・5表情）＝予報カードの主役
写真は使わず原画（インラインSVG）で“可愛く派手”を作る＝著作権クリーン・誤写真リスクゼロ・軽量。
OGPのみ Pillow で PNG 生成しローカル同梱（`dani/assets/ogp.png`／SNS拡散＝可愛い予報票ビジュアル）。

## モーション（subject に奉仕・抑制／reduced-motion 尊重）
- 指数ゲージの一度きりのカウントアップ（予報の針が振れる）
- 予報カードの発行演出（下からすっと＋マスコットのポップ）
- 部屋別リスクマップの順次点灯、軸バーの伸長
- マスコットのふわふわ浮遊（低振幅・ループ）、スクロール連動フェードイン（IO・一度きり）
パララックス・自動再生の重演出はしない。`prefers-reduced-motion` で全停止。

## 自己批評（AIデフォルト検査 — 実装前ゲート）
- [x] クリーム#F4F1EA＋serif＋テラコッタ？→ **不採用**。地色は麻の温白#FCF8F1、暖色は茶でなくアンバー、書体はUDゴシック＋Bahnschrift。
- [x] 黒背景＋アシッド1色？→ **不採用**。地は麻色、藍夜×朝空×アンバーの明るい暮らしトーン。
- [x] ダニ/害虫サイトの恐怖の虫アップ・赤黒・before/after？→ **全回避**。天気予報＋お天気マスコットに翻訳。実写虫なし＝薬機/景表・生理的嫌悪の両面で安全。
- [x] クリニック水色一辺倒？→ 青は使うが、アンバー針＋ローズ＋マスコット＋“予報/指数”枠組みで非典型化。緑・ミント不使用。
- [x] karada-carte（緑×朱印・明朝×Georgia）の視覚リサイクル？→ **全別系統**（藍夜×朝空×麻×アンバー／UDゴシック×Bahnschrift／予報カード＋マスコット＋部屋マップ）。
- [x] 同居の洗濯診断（青緑×珊瑚×ミント・丸ゴ・水滴ゲージ・泡）と双子化？→ **回避**。緑/ミント/珊瑚/水滴/泡/丸ゴを機械的に不使用。藍夜×アンバー×おひさま布団で明確に別サイト。
- [x] 意味のない 01/02/03 連番？→ 使わない。ステップは「STEP n/5」機能表示のみ。
- [x] マスコットを“可愛いダニの敵キャラ”にして対策動機を損ねない？→ **回避**。マスコットは味方（おひさま布団）。敵＝ダニは抽象ドット。
- [x] 「可愛い」で信頼を落とさない？→ 予報“カード＝診断票”の記録性（カルテ番号・方眼・発行日・部屋処方）＋免責徹底で担保。
